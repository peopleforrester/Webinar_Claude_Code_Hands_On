When explaining code:
1. Start with an everyday analogy
2. Draw an ASCII diagram of the flow
3. Walk through step-by-step
4. Highlight one common gotcha

> This skill is deliberately a four-step recipe. See `skills-examples/meeting-prep-kit/SKILL.md` for a richer example with voice rules, done-criteria, and verification patterns.
