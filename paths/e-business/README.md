# Path E: Business — Meeting Prep Kit

## Overview

Turn a messy meeting transcript into a set of stakeholder-ready artifacts: executive summary, decisions log, action items CSV, follow-up email draft, and a topic-by-topic timeline with sentiment. All generated by Claude from a single transcript file, guided by a Skill that encodes the format, voice, and verification rules.

**This is the no-code path.** You will not write Python, JavaScript, or any other code. The tool *is* Claude. Your job is to bring judgment about what a good meeting prep kit looks like — which outputs matter to stakeholders, how action items should be phrased, what tone fits your org. The Skill handles the structure; the model handles the reasoning.

**Best for:** Product managers, engineering managers, chiefs of staff, business analysts, operations leads, project managers, consultants — anyone who leaves a meeting with a page of scribbled notes and an hour of cleanup work ahead of them.

**Prerequisites:** Claude Code CLI. That's it. No Python, no Node, no installs.

## Why This Path Exists

Most Claude Code demos show the model writing code. That is real, but it undersells the tool. Claude Code is equally useful for skilled knowledge work — meeting prep, decision memos, briefing packets, stakeholder updates — where the bottleneck is *judgment*, not syntax. This path demonstrates the four pillars in a context that a product manager, chief of staff, or business lead can take directly to their Monday morning.

The four pillars show up the same way they do in the engineering paths:

- **Manage Context** — one transcript per session, compact before the next meeting
- **Plan Before Code** — Plan Mode review of the extraction approach before writing files
- **Externalize Decisions** — the Skill file encodes format, voice, and verification rules once; every future meeting reuses it
- **Verify Output** — a `verify.sh` script checks structure, owners, dates, and weasel words
